# Plan 7.1: Hot-Reload for PC Hub (debugpy)

## Objective
Speed up the development feedback loop by enabling the Python Hub to automatically restart when source files are modified, without manually killing and re-running `main_server.py`.

## Context
- hub/main_server.py

## Tasks

<task type="auto">
  <name>Install watchdog / add debugpy configuration</name>
  <files>hub/requirements.txt, hub/dev_server.sh</files>
  <action>
    - Add `watchdog` and `debugpy` to the Hub's requirements.
    - Create a dev launcher shell script `hub/dev_server.sh` that invokes `python -m debugpy --listen 5678 --wait-for-client hub/main_server.py` for attaching a debugger.
    - Additionally add a simple `watchmedo auto-restart` alias to trigger hot-reload on `.py` file changes in the `hub/` and `pc_hub/` directories.
  </action>
  <verify>cat hub/dev_server.sh</verify>
  <done>Dev server script exists and triggers hot-restart on file changes.</done>
</task>

## Success Criteria
- [ ] Code changes to the Hub are reflected in the running process within 2 seconds.
